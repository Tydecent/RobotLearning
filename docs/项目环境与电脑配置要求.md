## 一、项目概述

本赛项围绕**具身智能机器人**在编程、环境感知、运动控制等场景的应用展开：


| 模块  | 内容                        | 时长   | 权重  |
| --- | ------------------------- | ---- | --- |
| A   | 机器人仿真模型修复与验证（URDF、Gazebo） | 2 小时 | 20% |
| B   | 机器人操作与控制（脚本、动作接口调用）       | 3 小时 | 40% |
| C   | 机器人导航（地图、定位、路径规划、避障）      | 3 小时 | 40% |


**核心技术栈**：ROS 2 Humble · Gazebo 11 · Rviz2 · URDF · Nav2 · 中慧云启竞赛平台

## 三、系统环境配置教程

### 阅读说明

- 顺序：**Windows 宿主机 → VMware → Ubuntu → ROS 2 Humble**

- **中文链接**：各节「参考」及第四章项目链接中，原文链接后附有 `[中文：…]` 版本（掘金、Robook 中文文档、清华镜像站、Robotis 手册等），便于国内网络环境下查阅

### ROS 2 Humble 架构说明

Ubuntu 2204 官方 Tier 1 配套的 ROS 2 发行版为 **Humble**，与赛题各模块对应关系如下：


| 组件                  | 用途              | 对应模块   |
| ------------------- | --------------- | ------ |
| URDF / xacro        | 模型描述、编译与加载      | 模块 A   |
| Gazebo 11 gazeboros | 虚拟仿真加载与验证模型     | 模块 A、C |
| Nav2                | 地图加载、定位、路径规划、避障 | 模块 C   |
| Rviz2               | 传感器数据与导航可视化     | 模块 C   |
| rclpy / 节点通信        | 脚本编写、话题与服务调用    | 模块 B   |


**注意**：技术工作文件要求 Rviz **11.2.26 及以上**，在 ROS 2 生态中对应 **Rviz2 11.x**（Humble 默认配套）；Foxy 配套 Rviz2 8.x 且已停止维护，无法满足赛场版本要求。比赛文档示例兼涉 ROS Noetic（ROS1）与 Rviz2（ROS2），训练环境统一为 **Ubuntu 22.04 + ROS 2 Humble**；建议了解 ROS1 与 ROS2 的差异及基本切换方式。

### 3.6 安装 ROS 2 Humble（虚拟机内）

**参考**：[ROS 2 Humble 官方安装文档](https://docs.ros.org/en/humble/Installation/Ubuntu-Install-Debians.html) · [中文：清华 ROS2 镜像安装帮助](https://mirrors.tuna.tsinghua.edu.cn/help/ros2/) · [中文：Ubuntu 22.04 搭建 ROS 2 Humble 环境（掘金）](https://juejin.cn/post/7567190160461152291)

#### 建议安装方式

推荐使用清华镜像源手动安装，完整步骤见上方「清华 ROS2 镜像帮助」与掘金教程。

```bash
sudo apt install -y curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] https://mirrors.tuna.tsinghua.edu.cn/ros2/ubuntu $(source /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
sudo apt update && sudo apt install -y ros-humble-desktop
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

验证：

```bash
echo $ROS_DISTRO          # 应输出 humble
ros2 --help               # 应正常显示帮助信息
```

### 3.7 rosdep 与 colcon 工作空间（虚拟机内）

**参考**：[ROS 2 创建工作空间官方教程](https://docs.ros.org/en/humble/Tutorials/Beginner-Client-Libraries/Creating-A-Workspace/Creating-A-Workspace.html) · [中文：ROS2 colcon 工作空间入门（掘金）](https://juejin.cn/post/6966155161613893663) · [中文：colcon 编译工具说明（掘金）](https://juejin.cn/post/6987258016806469645)

`rosdep`是 ROS / ROS2 里用来自动查找并安装功能包依赖的工具。

#### 它解决什么问题

ROS 工作空间里有很多包（`package.xml`、`CMakeLists.txt` 等会声明依赖），这些依赖可能是：

- 系统 apt 包（如 `libopencv-dev`）
- 其他 ROS 包（如 `ros-humble-navigation2`）

手动一个个装很容易漏、版本也不统一。`rosdep` 会读这些声明，然后告诉你/帮你安装缺什么。

```bash
sudo apt install -y python3-rosdep python3-colcon-common-extensions
sudo rosdep init          # 若已初始化，忽略报错
rosdep update
```

下面这些命令用于创建并配置 ROS 2 工作空间（workspace），让后续自己写的/克隆的包能编译、运行，并在每次开终端时自动生效。

```bash
mkdir -p ~/ros2_ws/src && cd ~/ros2_ws
colcon build
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

[ROS2多空间管理指南](./ROS2多工作空间管理指南.md)

验证：

```bash
echo $ROS_DISTRO            # 应输出 humble
echo $COLCON_PREFIX_PATH    # 应含 ~/ros2_ws/install
```

### 3.8 Gazebo 11 与 ROS 2 集成（虚拟机内）

**参考**：[Gazebo Classic 官方 Ubuntu 安装说明](https://classic.gazebosim.org/tutorials?tut=install_ubuntu) · [Ubuntu 22.04 gazebo 包](https://packages.ubuntu.com/jammy/gazebo) · [gazebo_ros_pkgs 仓库](https://github.com/ros-simulation/gazebo_ros_pkgs) · [中文：URDF 在 Gazebo 中使用（Robook）](https://ros2docs.robook.org/humble/Tutorials/Intermediate/URDF/Using-a-URDF-in-Gazebo.html) · [中文：Gazebo 与 Rviz2 区别（掘金）](https://juejin.cn/post/7016940298106322980)

在 **Ubuntu 虚拟机里的任意终端目录** 都可以执行，**不依赖** `~/ros2_ws`，也**不必**先 `cd` 到某个工作空间。

**Ubuntu 22.04 说明**：从 Jammy 起，Open Robotics **不再**在 `packages.osrfoundation.org` 发布 Gazebo 11 二进制包；应使用 Ubuntu 官方 **universe** 源。包名与 Ubuntu 20.04 不同，但安装的仍是 **Gazebo Classic 11.x**（满足赛场要求）：

| Ubuntu 20.04 旧包名 | Ubuntu 22.04 正确包名 |
| ---------------- | ----------------- |
| `gazebo11`       | `gazebo`          |
| `libgazebo11-dev`| `libgazebo-dev`   |

```bash
# 若曾按旧教程添加过 OSRF 源，先移除（可选，避免 apt 警告）
sudo rm -f /etc/apt/sources.list.d/gazebo-stable.list

sudo add-apt-repository universe
sudo apt update
sudo apt install -y gazebo libgazebo-dev
sudo apt install -y ros-humble-gazebo-ros-pkgs ros-humble-gazebo-ros2-control
gazebo --version            # 应显示 11.x
```

仿真测试（两终端）：


| 终端  | 命令                                        |
| --- | ----------------------------------------- |
| 1   | `ros2 launch gazebo_ros gazebo.launch.py` |
| 2   | `ros2 topic list`（应见 Gazebo 相关话题）         |

| VMware 无法创建 64 位 VM   | 附录 A                                                   |
| `sudo rosdep init` 报错 | 已初始化，跳过                                                |
| `gazebo11` 无安装候选     | Ubuntu 22.04 改用 `sudo apt install gazebo libgazebo-dev`，见 3.8 |
| Gazebo 黑屏/闪退          | VM 启用 3D 加速                                            |
| 内存卡顿                  | 宿主机 64 GB，VM 分配 24 GB                                  |
| `ros2` 找不到            | 执行 `source /opt/ros/humble/setup.bash`                   |
| colcon build 失败       | 先 `rosdep install --from-paths src --ignore-src -r -y` |
| Rviz2 无导航数据           | 确认 Nav2 与仿真/传感器节点均已启动                                  |


### 附录 A：开启 CPU 虚拟化（故障排查）

**仅当** VMware 提示虚拟化未启用、无法创建 64 位 VM，或任务管理器显示「虚拟化：已禁用」时需要。

**参考**：[Microsoft 在 Windows 启用虚拟化](https://learn.microsoft.com/zh-cn/windows/wsl/tutorials/virtualization) · [中文：BIOS 开启虚拟化技术（掘金）](https://juejin.cn/post/7139822111840960543) · [中文：Win11 VT-x 故障排查（掘金）](https://juejin.cn/post/7240380161555972156)

1. 任务管理器 → 性能 → CPU → 确认状态
2. 若「已禁用」：重启进 BIOS（`Del`/`F2`/`F10`）
3. 启用 **Intel VTx / VTd** 或 **AMD SVM Mode** → `F10` 保存
4. 仍异常时（PowerShell 管理员）：`dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart` → 重启
