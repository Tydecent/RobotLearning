## 一、项目概述

本赛项围绕**具身智能机器人**在编程、环境感知、运动控制等场景的应用展开：


| 模块  | 内容                        | 时长   | 权重  |
| --- | ------------------------- | ---- | --- |
| A   | 机器人仿真模型修复与验证（URDF、Gazebo） | 2 小时 | 20% |
| B   | 机器人操作与控制（脚本、动作接口调用）       | 3 小时 | 40% |
| C   | 机器人导航（地图、定位、路径规划、避障）      | 3 小时 | 40% |


**核心技术栈**：ROS 2 Foxy · Gazebo 11 · Rviz2 · URDF · Nav2 · 中慧云启竞赛平台

## 三、系统环境配置教程

### 阅读说明

- 顺序：**Windows 宿主机 → VMware → Ubuntu → ROS 2 Foxy**
- 命令默认在 **Ubuntu 虚拟机终端**执行，标注「Windows 宿主机」的除外
- **CPU 虚拟化**：多数电脑出厂已开启，任务管理器 → 性能 → CPU 查看；仅 VMware 报错时见 [附录 A](#附录-a开启-cpu-虚拟化故障排查)
- **中文链接**：各节「参考」及第四章项目链接中，原文链接后附有 `[中文：…]` 版本（CSDN、鱼香 ROS、Nav2 中文网、博客园等），便于国内网络环境下查阅

### ROS 2 Foxy 架构说明

Ubuntu 20.04 官方配套的 ROS 2 发行版为 **Foxy**，与赛题各模块对应关系如下：


| 组件                     | 用途              | 对应模块   |
| ---------------------- | --------------- | ------ |
| URDF / xacro           | 模型描述、编译与加载      | 模块 A   |
| Gazebo 11 + gazebo_ros | 虚拟仿真加载与验证模型     | 模块 A、C |
| Nav2                   | 地图加载、定位、路径规划、避障 | 模块 C   |
| Rviz2                  | 传感器数据与导航可视化     | 模块 C   |
| rclpy / 节点通信           | 脚本编写、话题与服务调用    | 模块 B   |


> **注意**：比赛文档示例使用了ROS noetic（ROS1），但是又要求使用Rviz2（ROS2），因此我们安装ROS2 Foxy。但是要掌握ROS1 noetic的使用以及双版本的切换。

### 3.6 安装 ROS 2 Foxy（虚拟机内）

**参考**：[Foxy/Ubuntu 安装](https://docs.ros.org/en/foxy/Installation/Ubuntu-Install-Debians.html) · [中文：CSDN Ubuntu 20.04 安装 ROS2 Foxy](https://blog.csdn.net/weixin_42499608/article/details/117921337) · [中文：鱼香 ROS Foxy 安装文档](http://fishros.org/doc/ros2/foxy/Installation/Ubuntu-Install-Debians.html)

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install -y curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
sudo apt update
sudo apt install -y ros-foxy-desktop python3-argcomplete
```

配置环境变量（写入 `~/.bashrc`，新开终端自动生效）：

```bash
echo "source /opt/ros/foxy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

验证：

```bash
echo $ROS_DISTRO          # 应输出 foxy
ros2 --help               # 应正常显示帮助信息
```

### 3.7 rosdep 与 colcon 工作空间（虚拟机内）

**参考**：[创建 colcon 工作空间](https://docs.ros.org/en/foxy/Tutorials/Beginner-Client-Libraries/Creating-A-Workspace/Creating-A-Workspace.html) · [中文：CSDN 创建 ROS2 工作空间（Foxy）](https://blog.csdn.net/weixin_42499608/article/details/118782229) · [中文：CSDN ROS2 Foxy colcon 工作空间笔记](https://blog.csdn.net/qq_45701501/article/details/118573303)

```bash
sudo apt install -y python3-rosdep python3-colcon-common-extensions
sudo rosdep init          # 若已初始化，忽略报错
rosdep update

mkdir -p ~/ros2_ws/src && cd ~/ros2_ws
colcon build
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

验证：

```bash
echo $ROS_DISTRO            # 应输出 foxy
echo $COLCON_PREFIX_PATH    # 应含 ~/ros2_ws/install
```

### 3.8 Gazebo 11 与 ROS 2 集成（虚拟机内）

**参考**：[Gazebo 11 安装](https://classic.gazebosim.org/tutorials?tut=install_ubuntu&ver=11.0) · [gazebo_ros_pkgs（ROS 2）](https://github.com/ros-simulation/gazebo_ros_pkgs) · [中文：CSDN Ubuntu 安装 ROS/ROS2 + Gazebo](https://jszn.csdn.net/6a053f720a2f6a37c5aa0c45.html) · [中文：CSDN 使用 Gazebo 加载 URDF（鱼香 ROS）](https://blog.csdn.net/qq_27865227/article/details/132506784)

```bash
sudo sh -c 'echo "deb http://packages.osrfoundation.org/gazebo/ubuntu-stable $(lsb_release -cs) main" > /etc/apt/sources.list.d/gazebo-stable.list'
wget https://packages.osrfoundation.org/gazebo.key -O - | sudo apt-key add -
sudo apt update
sudo apt install -y gazebo11 libgazebo11-dev
sudo apt install -y ros-foxy-gazebo-ros-pkgs ros-foxy-gazebo-ros2-control
gazebo --version            # 应显示 11.x
```

仿真测试（两终端）：


| 终端  | 命令                                        |
| --- | ----------------------------------------- |
| 1   | `ros2 launch gazebo_ros gazebo.launch.py` |
| 2   | `ros2 topic list`（应见 Gazebo 相关话题）         |

### 常见问题

| 内存卡顿                  | 宿主机 64 GB，VM 分配 24 GB                                  |
| `ros2` 找不到            | 执行 `source /opt/ros/foxy/setup.bash`                   |
| colcon build 失败       | 先 `rosdep install --from-paths src --ignore-src -r -y` |
| Rviz2 无导航数据           | 确认 Nav2 与仿真/传感器节点均已启动                                  |


### 附录 A：开启 CPU 虚拟化（故障排查）

**仅当** VMware 提示虚拟化未启用、无法创建 64 位 VM，或任务管理器显示「虚拟化：已禁用」时需要。

**参考**：[Microsoft 启用虚拟化](https://support.microsoft.com/en-us/windows/enable-virtualization-on-windows-c5578302-6e43-4b4b-a449-8ced115f58e1) · [中文：CSDN CPU 虚拟化开启教程](https://blog.csdn.net/weixin_41267955/article/details/141291122) · [中文：VT 开启图文教程（驱动人生）](https://www.160.com/article/10921.html)

1. 任务管理器 → 性能 → CPU → 确认状态
2. 若「已禁用」：重启进 BIOS（`Del`/`F2`/`F10`）
3. 启用 **Intel VT-x / VT-d** 或 **AMD SVM Mode** → `F10` 保存
4. 仍异常时（PowerShell 管理员）：`dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart` → 重启
